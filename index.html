A History of Recorded Sound

Introduction

New technologies did not come to exist in a vacuum; they instead come from a lineage of many technologies from the past. Haas (1999) speaks of the importance of complicating existing, overly simplistic ideas that people have about technological progress. One common idea is that of simple replacement, “which assumes that old and new technologies are clearly differentiated from each other, both theoretically and in practice, and that new technologies simply replace older, obsolete technologies” (p.210). However, many technologies complement each other and can even be used alongside each other. This is true with music-- for example, the use of CDs in a car, writen music books in front of a piano, and MP3s on a laptop. The second simplistic idea is that of straightforward progress, which is the idea that new technologies are inherently better than old technologies. While improvements can be made, it is important to think critically about these changes. Sometimes, when you gain something, you lose something else.

The History of <audio> Element

HTML stands for HyperText Markup Language, which a common programming language for the creation of websites. HTML uses elements, which are sometimes referred to as tags, in order to describe and style content on a web page. This website will focus on the history of the <audio> element, which is used to add audio content to websites.

The <audio> element was added in HTML5. Before that, websites needed to use plugins such as Flash and Silverlight to display audio or video content on websites. However, these had problems with security and did not always work well with HTML and CSS elements. Therefore, the native HTML audio element was added in HTML5 to overcome these issues (cite Mozilla website).

Syntax and Attributes

Here is some example code with the <audio> element:

<audio controls> <source src="flexidrum.wav" type="audio/wav" autoplay="true" loop="true"> Your browser does not support the HTML5 audio element. Here is a <a href="flexidrum.wav">link to the audio</a> instead. </audio>

Attributes

controls:
This uses the browser's default audio control interface. You can also build your own interface using JavaScript. Minimum requirements for custom controls are the ability to start, stop, and adjust the volume.

src:
The source attribute contains a path to the audio you wish to embed.
type:
descrives the file format

fallback content:
The paragraph inside the audio tags will display if the browser is older and cannot display the <audio> element.

Basic Audio Formats

According to W3 Schools, the 3 supported audio formats in HTML5 are MP3, WAV, and OGG.

.MP3
MP3 is the best-known compressed file format. However, it is a liscenced format which requires its distributors to pay 2% of all revenue over $100k. Also, game creators need to pay a licence fee if they distribute over 5,000 copies. (cite: construct.net article. May want better source.)
.WAV
WAV is an uncompressed media type. The files are often very large and can have long loading times.
.OGG
Ogg is similar to MP3 but does not have any patents or required royalty fees. However, it is not currently supported on Internet Explorer or Safari.

Analysis and Conclusions

Is the audio element an improvement on past technologies? This depends on the context. Digital audio players offer a great deal of convienience to people using devices with internet access such as laptops, tablets, and phones. When the content is downloaded, it can be played from anywhere with enough battery power. However, the audio tag itself involves streaming content from online; therefore, it's only viable if you have an internet connection. Written music is still the standard for much of music education (cite?) since it provides necessary information without the distractions inherent in a device connected to the internet. In addition, older media formats such as CD-ROMs may be more convienient when riding in older cars. While digital music is currently the most common format in the modern world, it is not a simple replacement for older formats-- rather, they are used alongside each other in different contexts.
